## What to do on Mac (pick up here)

### Step 1: Pull the branch

```bash
git fetch origin
git checkout feature/ios-appstore-submission
git pull
```

### Step 2: Install CocoaPods (if not already installed)

```bash
brew install cocoapods
# OR: sudo gem install cocoapods
```

### Step 3: Run expo prebuild

```bash
cd frontend
npx expo prebuild --platform ios --clean
```

This generates the `ios/` Xcode project from `app.json`. It will:
- Create `ios/Bookshelf.xcworkspace`
- Burn `com.buffingchi.bookshelf` into the Xcode project
- Generate `ios/Bookshelf/PrivacyInfo.xcprivacy` from the `privacyManifests` block

### Step 4: Install CocoaPods dependencies

```bash
cd ios
pod install
cd ../..
```

### Step 5: Verify the privacy manifest

```bash
cat frontend/ios/Bookshelf/PrivacyInfo.xcprivacy
```

Should contain XML with `NSPrivacyAccessedAPICategoryUserDefaults` and reason `CA92.1`. If the file is missing or empty, see the Fallback section below.

### Step 6: Verify bundle ID

```bash
grep -r "com.buffingchi.bookshelf" frontend/ios/
```

Should show hits in `project.pbxproj` and `Info.plist`.

### Step 7: Run Jest tests

```bash
cd frontend
npx jest
```

All tests should pass — no TypeScript source was changed.

### Step 8: Commit and push the ios/ directory

```bash
git add frontend/ios/
git commit -m "feat(ios): add expo prebuild bare workflow ios/ directory

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
git push
```

## After the ios/ commit — Apple account setup

### Step 1: Enroll in Apple Developer Program
1. Go to `https://developer.apple.com/programs/enroll/`
2. Sign in with your Apple ID (wcmchenry3@gmail.com)
3. Select **Individual / Sole Proprietor**
4. Enter legal name exactly as on government ID
5. Pay $99 USD/year
6. Enable 2FA on your Apple ID first — required for enrollment
7. Approval is usually same-day for individuals (up to 48h)

### Step 2: Accept agreements in App Store Connect
1. Go to `https://appstoreconnect.apple.com`
2. Agreements, Tax, and Banking → accept Paid Applications Agreement
3. Fill in tax + banking info (required even for free apps)

### Step 3: Create App ID + App Store Connect record
1. **Developer portal** → Certificates, Identifiers & Profiles → Identifiers → "+"
   - Type: App IDs → App
   - Description: `Bookshelf`
   - Bundle ID: **Explicit** → `com.buffingchi.bookshelf` (copy-paste this exactly)
   - Save
2. **App Store Connect** → My Apps → "+" → New App
   - Platform: iOS
   - Name: `Bookshelf` (if taken, try `Bookshelf: Reading Tracker`)
   - Bundle ID: select `com.buffingchi.bookshelf`
   - SKU: `bookshelf-ios-001`

### Step 4: Add privacy policy URL to App Store Connect
- App Information → Privacy Policy URL → `https://wcmchenry3-stack.github.io/book_app/privacy.html`
- Verify the URL loads in a browser first

### Step 5: Build and archive in Xcode
1. Open `frontend/ios/Bookshelf.xcworkspace` in Xcode
2. Signing & Capabilities → check **Automatically manage signing** → Team: your account
3. Bundle Identifier must show `com.buffingchi.bookshelf`
4. Destination → **Any iOS Device (arm64)** — NOT a simulator
5. **Product → Archive** (3–10 min)
6. Organizer → **Distribute App → App Store Connect → Upload**
7. Leave all defaults → Upload
8. Processing takes 5–30 min; email confirmation when done

### Step 6: Set up TestFlight
1. App Store Connect → your app → TestFlight tab
2. Find the build under iOS Builds
3. If yellow warning icon: click → answer encryption questions (or it's pre-answered by `ITSAppUsesNonExemptEncryption: false`)
4. Internal Testing → "+" → add testers by Apple ID email
5. Testers must be in Users and Access first
6. Enable build for the group → testers get TestFlight invite email

## Fallback: PrivacyInfo.xcprivacy not generated by prebuild

If `ios/Bookshelf/PrivacyInfo.xcprivacy` is missing after prebuild, create it manually:

```bash
cat > frontend/ios/Bookshelf/PrivacyInfo.xcprivacy << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>NSPrivacyAccessedAPITypes</key>
  <array>
    <dict>
      <key>NSPrivacyAccessedAPIType</key>
      <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
      <key>NSPrivacyAccessedAPITypeReasons</key>
      <array>
        <string>CA92.1</string>
      </array>
    </dict>
  </array>
  <key>NSPrivacyCollectedDataTypes</key>
  <array/>
  <key>NSPrivacyTracking</key>
  <false/>
  <key>NSPrivacyTrackingDomains</key>
  <array/>
</dict>
</plist>
EOF
```

Then add it to the Xcode project by dragging it into the `Bookshelf` group in Xcode's file navigator (ensure "Add to target: Bookshelf" is checked).
